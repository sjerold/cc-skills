# 大唐三公六部多Agent协作系统

大唐贞观年间架构，三公决策、六部执行，支持任意任务的多Agent协作。

## 核心特性

- **任务通用性**：动态适配任意类型任务
- **多模态支持**：输入（文本/图片），输出（文档/图表/Word）
- **分层架构**：三公决策 → 六部执行
- **并行执行**：兵贵神速，动态并发控制
- **实时响应**：动态响应用户中途输入
- **进度监控**：整体进度、Agent健康度、排队状态

## 三公六部架构

```
【三公】 决策与监管层
  太师府（房玄龄）→ 运筹帷幄，战略规划
  太傅府（魏征）  → 审核批驳，质量把关
  太保府（长孙无忌）→ 监察百官，调度六部

【六部】 执行层
  吏部（裴行俭）→ 状态追踪
  户部（刘晏）  → 资源管理/输入处理
  礼部（贺知章）→ 文档生成/输出呈现
  兵部（李靖）  → 任务攻坚/General Worker
  刑部（狄仁杰）→ 质量审核
  工部（阎立德）→ 工程构建/Code Builder
```

## 使用方式

```bash
# 标准模式
/t 分析项目架构并生成文档

# 快速模式
/t 分析代码 --mode=quick

# 严格模式
/t 开发关键功能 --mode=strict

# 讨论议题
/t 讨论 数据库选型方案
```
