# 大唐三公六部系统

本技能通过**三公六部**架构实现多Agent协作，采用大唐贞观年间历史人物命名，
支持任意类型任务的多模态处理。

## 核心特性

- **任务通用性**：动态适配任意类型任务
- **多模态支持**：输入（文本/图片），输出（文档/图表/Word）
- **分层架构**：三公决策 → 六部执行
- **并行执行**：兵贵神速，动态并发控制
- **实时响应**：动态响应用户中途输入
- **进度监控**：整体进度、Agent健康度、排队状态

## 工作流程

### Phase 1: 太师府规划

1. **接收圣旨**：分析用户任务意图（支持多模态输入）
2. **判断任务等级**：
   - 闲聊 → 直接回复
   - 简单 → 单Agent执行
   - 中等 → 串行工作流
   - 复杂 → 完整三公六部流程
3. **制定奏折**：输出 `workspace/zouzhang.json`

### Phase 2: 太傅府审核

1. **审核奏折**：检查规划可行性
2. **批驳或通过**：不通过则返回太师重新规划（最多3次迭代）
3. **移交太保**：审核通过后进入执行阶段

### Phase 3: 太保府调度

1. **解析奏折**：读取任务分解和依赖关系
2. **查询吏部**：获取空闲Agent状态
3. **调度六部**：按依赖关系并行启动执行Agent
4. **监控进度**：定期输出进度报告，处理异常

### Phase 4: 六部执行

根据任务类型调度对应部门：
- **户部**：输入解析（图片、文件、数据）
- **兵部**：分析、搜索、统计（General Worker）
- **工部**：代码编写、工程构建（Code Builder）
- **刑部**：质量检查、问题诊断
- **礼部**：文档生成、图表绘制
- **吏部**：状态记录、Agent池管理

### Phase 5: 太傅府验收

1. **验收成果**：检查各部输出质量
2. **整改或通过**：不通过则要求重做
3. **汇总圣卷**：生成最终输出

## 任务分配原则

### 部门职责速查

| 部门 | 角色 | 任务类型 | 示例 |
|------|------|---------|------|
| 户部 | 输入处理 | 解析文件、图片、数据 | "解析这张架构图" |
| 兵部 | General Worker | 分析、搜索、统计 | "分析代码结构" |
| 工部 | Code Builder | 编写代码、修改文件 | "实现登录功能" |
| 刑部 | 质量审核 | 检查、验证、诊断 | "检查代码质量" |
| 礼部 | 输出生成 | 文档、图表、Word | "生成分析报告" |
| 吏部 | 状态管理 | Agent状态、任务分配 | 状态查询 |

### 兵部 vs 工部区分

**口诀：分析找兵部，写码找工部**

```yaml
兵部（General Worker）:
  ✅ 分析代码、统计数据、搜索信息
  ✅ 对比差异、推理逻辑、生成建议
  ❌ 编写/修改代码文件

工部（Code Builder）:
  ✅ 创建新代码、修改现有代码
  ✅ 写代码、改配置、重构模块
  ❌ 纯分析任务（交给兵部）
```

## 使用方式

```bash
# 标准模式（默认）
/t 分析项目架构并生成文档

# 快速模式，跳过部分审核
/t 分析代码 --mode=quick

# 严格模式，每个阶段都审核
/t 开发关键功能 --mode=strict

# 设置并发上限
/t 复杂任务 --max-concurrent=5
```

## 执行入口

当用户发送 `/t` 命令时，按以下步骤执行：

1. **启动太师府Agent**：分析任务，生成奏折
2. **启动太傅府Agent**：审核奏折
3. **启动太保府Agent**：调度六部执行
4. **监控进度**：定期输出状态
5. **太傅验收**：最终质量把关
6. **输出圣卷**：返回最终结果
