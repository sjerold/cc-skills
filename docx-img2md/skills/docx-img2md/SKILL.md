# Docx图片转Markdown 技能

将包含图片（截图/扫描件）的 docx 文档转换为 Markdown 格式。

## 使用场景

- docx 文档内容全部是截图或扫描件
- 需要将图片中的文字提取出来转为可编辑的 Markdown
- 流程图等图形类图片需要保留图片引用

## ⚠️ 执行环境强制要求（必须遵守）

**所有 Python 脚本必须使用 PowerShell 执行，禁止使用 Bash！**

### 原因
1. **中文路径问题**：docx 文件路径通常包含中文（如 `C:\Users\admin\Documents\文档.docx`），Bash 环境无法正确处理
2. **cmd /c 命令问题**：在 Bash 中执行 `cmd /c` 只会输出 Windows 版本信息，无法实际执行命令
3. **conda 环境问题**：只有 PowerShell 能正确激活 conda 环境并执行 Python 脚本

### 正确做法
```powershell
# ✅ 正确：使用 PowerShell 工具执行
cmd /c "call conda activate dsbot_env && python <脚本路径> <参数>"
```

### 错误做法
```bash
# ❌ 错误：使用 Bash 工具执行（会导致中文路径乱码或命令无法执行）
cmd /c "call conda activate dsbot_env && python <脚本路径> <参数>"
```

**全程禁止**：
- ❌ 禁止在任何 Bash 命令中执行本 Skill 的 Python 脚本
- ❌ 禁止尝试用 Bash 的语法转义中文路径
- ❌ 禁止跳过 PowerShell 环境直接调用 python

### 2. 批次识别图片写入 md

**⚠️ 使用外部 LLM API 识别图片（kimi-k2-5）**

图片识别通过 `external_ocr.py` 脚本调用 kimi API 完成。

**⚠️ 强制要求：单线程批次处理**

- **必须单线程**：主流程必须单线程顺序处理，每批处理完再处理下一批
- **禁止多线程**：绝对禁止多线程并发处理多批图片
- **批次内部并行**：脚本内部每批最多2张图片并行（线程池），但批次之间必须串行

**环境变量配置（一次性）**：

```powershell
$env:SP_TOKEN = "your-api-key"
```

- ❌ 禁止多线程并发处理多批图片
- ❌ 禁止同时启动多个 external_ocr.py 进程

```
[纯文字] 或 [混合] 或 [纯图形]
OCR 文字内容...

```

**图片识别提示词模板（每次调用必须传入）**：

```
请识别这张图片，并按以下准则处理：

【图片来源】docx 文档中的截图/扫描件
【处理方式】
1. 判断图片类型：
   - 纯文字类（截图/扫描件/文档正文）：直接 OCR 识别所有文字，按从上到下、从左到右顺序输出
   - 混合类（文字 + 流程图/图表）：OCR 识别文字，同时保留原图引用
   - 纯图形类（纯流程图/架构图，无实质文字）：仅返回 "![image](pic/<图片文件名>)"

2. 图片分类必须经过二次确认：
   - 如果初步判断含图形元素，必须确认图形占比是否 > 50%
   - 以文字/表格为主、只有少量装饰图标的 → 归类为纯文字类

3. OCR 文字识别时跳过：
   - 纯数字页码：1, 12, - 3 -
   - 版本号：V01xxxxx_20251124
   - 分隔线：---, ———
   - 罗马数字页码：I, II, III
   - 日期格式：2025-11-24

4. 跨页表格处理：
   - 如果表格被截断（底部无完整边框线），只识别当前页的数据行
   - 不要重复表头

【输出格式】
第一行输出图片类型：[纯文字] 或 [混合] 或 [纯图形]
第二行开始输出识别内容：
- 纯文字类：直接输出识别到的文字内容
- 混合类：先输出文字内容，最后单独一行输出 [需要原图引用]
- 纯图形类：只输出 [纯图形]

【禁止行为】
- 不要编写任何脚本
- 不要调用 OCR 库
- 只处理这一张图片
```

**写入 md 文件时**：
- md 文件路径：`<docx同目录>/文字版/<文档名>/<文档名>.md`
- 用 `Write` 工具首次写入标题，后续用 `Edit` 工具追加内容
- 图片引用路径相对于 md 文件为 `pic/xxx.png`
- **根据脚本返回的类型处理**：
  - `[纯文字]`：直接追加 OCR 内容
  - `[混合]`：追加 OCR 内容，最后加 `![image](pic/xxx.png)`
  - `[纯图形]`：只追加 `![image](pic/xxx.png)`

### 3. 跨页表格处理（重要）

**问题**：同一个表格跨越多张图片时，每张图片只识别自己那部分，会导致：
- 表格被拆成多个独立表格块
- 表格行顺序错乱
- 表头重复出现

**处理方法**：
1. 识别到图片包含表格时，判断是否为跨页表格：
   - 图片底部表格没有完整的底边框线（表格被截断）
   - 图片顶部表格没有完整的顶边框线（表格从上一页延续）
   - 页眉/页脚出现在表格中间位置
   - 表格列数、列宽与前后图片一致

2. **如果确认是跨页表格**：
   - 第一页（有表头）：正常写入完整表格（表头+数据行）
   - 中间页（无表头，只有数据行）：将识别的数据行追加到上一个表格的末尾，**不要重复表头**
   - 最后一页（表格结束）：同样追加数据行到表格末尾

3. **Markdown 表格追加行的方法**：
   - 用 `Edit` 工具找到 md 文件中上一个表格的最后一行（`|---|---|---|` 分隔行下面的最后一行数据）
   - 在该行后面追加新的数据行
   - 确保列数一致

4. **完成所有图片后**：
   - 通读 md 文件，检查是否有断裂的表格
   - 将断裂的表格合并为一个完整表格

### 4. 完成

确认 md 文件包含所有图片的识别结果，图片引用路径正确，表格完整无断裂。

## 图片命名规则

格式：`image_001.png`

- 使用英文前缀 `image` + 三位序号（从 001 开始）
- 统一转为 PNG 格式
- **禁止使用中文或特殊字符作为文件名**，避免 Markdown 导入失败

## 图片分类规则（重要）

### 第一步：初步判断

如果图片包含以下特征，**初步标记**为含图形元素：
- 箭头连接线、方框流程图
- 树状图、架构图、时序图
- UML 图、状态机图
- 饼图、柱状图等数据可视化图

纯表格、段落文字的图片初步判断为文字类。

### 第二步：二次确认（必须）

**初步标记为含图形的图片，必须进行二次确认**，因为很多以文字为主的图片会被误判。

二次确认标准：
- 图片中**图形元素占比超过 50%** → 确认为含图形
- 图片以**文字/表格为主，只有少量图标或装饰性图形** → 重新归类为文字类
- **带少量标注的表格** → 归类为文字类
- **带少量图标的纯文本段落** → 归类为文字类

### 第三步：根据分类处理

| 分类 | 处理方式 |
|------|---------|
| 纯文字类 | OCR 识别文字，写入 md |
| 混合类（文字+图形） | 用 `crop_graphics.py` 裁剪图形区域 + OCR 文字标注 + 裁剪图引用 |
| 纯图形类 | 仅保留原图引用 `![image](pic/xxx.png)` |

### 混合类图片的处理流程

混合类图片（既有文字又有流程图/图表）使用 `crop_graphics.py` 脚本处理：


1. PaddleOCR 检测所有文字区域及边界框

3. 识别图注文字（"图1 xxx"）确定图形下边界

**跳过的文字模式**：
- 纯数字（页码）：`1`, `12`, `- 3 -`

- 罗马数字页码：`I`, `II`, `III`
- 日期格式：`2025-11-24`, `2025/11/24`

**输出文件**：
- `xxx_graphic.png` — 裁剪后的纯图形部分（去除文字区域）
- `xxx_annotated.json` — 文字区域标注（含坐标、文字内容）

```

```

文字内容按从上到下、从左到右的顺序排列，图形引用放在最后。

## Markdown 换行规范（重要）

写入 md 文件时，必须遵守以下换行规则：

1. **列表项换行**：
   - a)、b)、c)... 或 1)、2)、3)... 等列表项，每项末尾加 `<br>` 强制换行
   - 错误：每项用普通换行（预览器可能合并为一行）
   - 正确：`a) 内容；<br>` `b) 内容；<br>` `c) 内容；`

2. **段落换行**：
   - 普通段落之间用一个空行分隔（两个换行符）
   - 同一段落内的换行用 `<br>` 标签

3. **表格单元格内的换行**：
   - 用 `<br>` 标签，不要用普通换行

4. **标题和正文**：
   - 标题行和正文之间空一行

5. **表格格式**：
   - 表头行：`| 列1 | 列2 | 列3 |`
   - 分隔行：`|------|------|------|`
   - 数据行：每行独立，`| 值1 | 值2 | 值3 |`
